# ЧАСТЬ 0 — ГЛАВНОЕ РЕШЕНИЕ (без него нельзя начинать)

## Проблема: 3 разных дизайна в референсах
Твои 10 картинок — из трёх итераций продукта:
- 4 страницы: бренд «AI Beauty Concierge», AI = «Алина», светлый сайдбар
- 3 страницы: бренд «BeautySaaS», AI = «Администратор», светлый сайдбар
- 1 страница (Активность): бренд «Тестовый салон», AI = **SERA**, **тёмный сайдбар** ✅

## Решение (рекомендую жёстко)
**Канон = образец №10 (Активность SERA).** Причины:
1. Это новейшая итерация
2. Единственная с орбом SERA
3. Тёмно-зелёный сайдбар `#10382F` совпадает с твоими токенами в CLAUDE.md
4. SERA — финальное имя бренда (Алина запрещена)

**Что делаем:** все 10+ страниц приводим к ОДНОМУ виду:
- Тёмно-зелёный сайдбар с орбом SERA (как img 10)
- Кремовый контент `#EFE9DD`, белые карточки
- Везде SERA, нигде Алина/Администратор/AI Beauty
- Один логотип, одни токены (`tokens.css`)

Остальные 8 референсов использую **только как карту функций** (какие поля и данные на странице), стиль — переделываю.

# ЧАСТЬ 3 — ЕДИНАЯ СТРУКТУРА ВСЕХ СТРАНИЦ

Каждая admin-страница строится по одному каркасу:

```
┌─────────────┬──────────────────────────────────────────┐
│             │  PAGE HEADER                              │
│  SIDEBAR    │  [Заголовок Cormorant] [подзаголовок]     │
│  (тёмный)   │                          [Primary action] │
│             ├──────────────────────────────────────────┤
│  • орб SERA │  KPI STRIP (4-5 метрик, единый компонент) │
│  • навигация├──────────────────────────────────────────┤
│  • SERA CTA │  FILTERS BAR (поиск + фильтры + сортировка)│
│  • тариф    ├──────────────────────────────────────────┤
│  • выход    │  CONTENT (карточки/таблица/график)        │
│             │                            [RIGHT RAIL?]  │
│             ├──────────────────────────────────────────┤
│             │  AI-HELPER BANNER (как SERA помогает...)  │
└─────────────┴──────────────────────────────────────────┘
```

**Единые компоненты для ВСЕХ страниц** (строим один раз, используем везде):
1. `<SeraSidebar>` — тёмный, орб, навигация, тариф-countdown
2. `<PageHeader title subtitle action>` — шапка
3. `<KpiStrip items[]>` — метрики (уже есть `KPIStrip`!)
4. `<FiltersBar search filters sort>` — панель фильтров
5. `<AiHelperBanner items[]>` — нижний баннер «как SERA помогает»
6. `<DataCard>` / `<DataTable>` — контент (с mobile-вариантом)
7. `<RightRail>` — правая колонка (мини-календарь, графики, советы)
8. `<EmptyState orb cta>` — пустые состояния
9. `<SeraOrb state size>` — орб (13 состояний)

Если эти 9 компонентов сделаны по `tokens.css` — все страницы автоматически в одном стиле.
